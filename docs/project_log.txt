KNOWN ISSUES (to address in future versions)
--------------------------------------------------------------------------------

  1. THREAD SAFETY — Medium priority
     The _heartbeat() thread and Flask request handlers both access
     self.filing, self.processor, etc. without a threading.Lock().
     Risk: race condition under heavy traffic. For demo/low-traffic: acceptable.
     Fix: Add threading.Lock() and wrap all shared-state access.

  2. HOST BINDING — Low priority for now
     app.run(host="0.0.0.0") means other devices on the same network can
     reach the server. For personal use: fine. For public deployment: restrict
     to host="127.0.0.1" or add authentication.

  3. NO PERSISTENT STORAGE — Medium priority
     All state resets when the server restarts (conversation log, node history).
     Fix: Add SQLite database for persistence.

  4. SIMULATED METRICS — By design for now
     All metrics (resonance, performance, entropy) are generated randomly.
     Future: connect to real data sources.

  5. AGENT RESPONSES ARE TEMPLATE-BASED — By design for now
     Agents respond using pre-built templates + random values.
     Future upgrade: connect to real Claude AI API for genuine responses.
