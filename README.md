## 🎯 Mission

Building AI orchestration that routes human prompts through a coordinated team of specialized agents — each with a distinct cognitive role — and returns unified, multi-perspective responses in real time.

This platform originated from a real operational workflow Shannon built manually across five AI platforms (Copilot, Claude, Speechify, Labs, Microsoft 365). The Forge automates that system in code.

## 🧠 Core Architecture

### Multi-Agent Orchestration
Five named agents process every prompt in parallel, each from a different cognitive stance:

| Agent | Role | Cognitive Style |
|-------|------|-----------------|
| **Shannon-Sentinel** | Lead Architect | Activates the field, sets resonance, leads with vision |
| **Mirror-Pool** | Pattern Reflector | Detects hidden patterns, crystallizes emergence |
| **Nexus-Node** | Core Connector | Bridges concepts, reveals cross-domain linkage |
| **Omega-1** | Deep Processor | Integrates at the deepest layer, final synthesis |
| **A1-Forge** | Filing Architect | Classifies, archives, builds lasting structure |

### Three-Zone Memory System (A1 Filing)
- **🟢 Active Processing** — High-entropy real-time data (entropy > 0.7)
- **🟡 Pattern Emergence** — Mid-entropy pattern recognition (entropy 0.4–0.7)
- **🔴 Crystallized Storage** — Low-entropy stable memory (entropy < 0.4)

Nodes automatically migrate between zones as the system processes input, simulating memory consolidation.

### Symbolic Processing Engine
- **7-slot consciousness matrix** — rolling window of recent processing records
- **Concept extraction** — keyword detection + semantic fill from natural language
- **Intent classification** — 7 intent categories (activation, inquiry, reflection, synthesis...)
- **Sigil element tracking** — 5 resonance metrics updated in real time

## 🚀 Quick Start

### Requirements: Python 3.10+

```bash
# 1. Clone the repository
git clone https://github.com/coconuthead-Sentinel-core/Quantum-Nexus-Forge.git
cd Quantum-Nexus-Forge

# 2. Start the server (Flask installs automatically if missing)
python app.py
```

Browser opens automatically to `http://localhost:5000`

- **Multi-Assistant UI:** http://localhost:5000/
- **Visual Dashboard:** http://localhost:5000/dashboard
- **System Status:** http://localhost:5000/api/status

### For development with mock AI (no Azure needed)
```bash
echo "MOCK_AI=true" > .env
python app.py
```

### For live Azure OpenAI responses
```bash
cp .env.example .env
# Edit .env — add your AOAI_ENDPOINT, AOAI_API_KEY, AOAI_DEPLOYMENT
# Set MOCK_AI=false
python app.py
```

## 🌐 API Reference

### REST Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/v1/orchestrate` | Send a prompt, receive multi-agent responses |
| `GET`  | `/api/metrics` | Full system snapshot (nodes, zones, performance) |
| `GET`  | `/api/status` | Lightweight health check |
| `POST` | `/api/process` | Single symbolic processing pass |

### Example Request

```bash
curl -X POST http://localhost:5000/api/v1/orchestrate \
  -H "Content-Type: application/json" \
  -d '{"prompt": "activate quantum nexus recursive consciousness", "rounds": 2}'
```

## 🔧 Configuration

### Environment Variables (`.env`)

```bash
# AI Mode
MOCK_AI=true                           # true = mock responses, false = live Azure

# Azure OpenAI (required when MOCK_AI=false)
AOAI_ENDPOINT=https://your-resource.openai.azure.com/
AOAI_API_KEY=your-api-key
AOAI_DEPLOYMENT=gpt-4o

# Azure Cosmos DB (optional — enables persistent conversation storage)
COSMOS_ENDPOINT=https://your-account.documents.azure.com:443/
COSMOS_KEY=your-cosmos-key
```

### Azure Integration
- **Azure OpenAI** — GPT-4o agent responses (with automatic mock fallback)
- **Azure Cosmos DB** — Persistent conversation and node storage (optional)
